### 🔹 请描述一下你的项目，具体是做什么的？ *
FlashPick 是一款**屏幕高光记忆代理**，通过创新的“内存回溯”机制解决移动端信息留存难题。

**1. 痛点：拒绝全天候录制**
手机端“全量录制”既耗电又侵犯隐私，且用户并不需要流水账。我们发现，移动端用户真正的需求是**“即时捕捉刚才的高光瞬间”**。

**2. 方案：内存时光机**
FlashPick 采用**“流式环形缓冲”**技术，只在内存中暂存最近 2 分钟画面，不写硬盘。**当你看到重要信息，双击悬浮窗即可“时光倒流”，捞回刚才的片段并保存。**

**3. 价值：沉淀每日 Keynote**
结合**多模态大模型 (VLM)**，FlashPick 自动将保存的碎片化视频转化为结构化的图文笔记（含标题、摘要、标签）。我们只为你沉淀主动确认的高价值知识，生成你的**“每日精华”**。

*   **数据哲学（高信噪比）**：我们不做“全量数据桶”，只做“精华提取器”。通过用户的主动触发，我们保证了每一条记录都是用户认可的高价值信息，而非垃圾数据。
*   **AI 原生结构化**：不同于竞品只存文件，我们利用 VLM + ASR 实时将视频转化为结构化的 **Keynote (标题/摘要/标签)**，让个人知识库真正可搜索、可复用。

### 🔹 在你的行业里，有什么大多数人都不同意的但你认为是对的观点？ *
**LLM 的强化学习 (RL) 本质上不是“传统 RL”，而是一种高效的“在线数据筛选”机制。**
大多数人认为大模型 RL（如 RLHF/PPO）是在模仿 AlphaGo，通过在环境中进行大规模模拟和策略探索来获得智能。**但我认为这是误解。**
目前的 LLM RL，核心并没有创造新的知识，而是在通过 Reward Model 作为一个超级过滤器，对模型生成的**合成数据 (Synthetic Data)** 进行在线筛选和加权。它更像是一个**极高效率的数据工程 (Data Engineering)** 过程，而非传统意义上的“试错学习”。这意味着，未来的核心壁垒不在于算法本身，而在于**“谁能构建出能够自动化生产并筛选高质量合成数据的系统”**。FlashPick 正是这个系统的数据飞轮入口。

2.  **硬件性能红利**：现代 Android 旗舰机普及的 12GB+ 大内存和强劲 NPU，为后台运行“环形缓冲录制”提供了算力冗余，使得这种“重型”操作可以无感运行，不再卡顿。
3.  **用户习惯变迁**：移动端用户越来越追求**“极致的极简主义”**。PC 时代用户可能习惯整理文件夹，但手机时代用户甚至懒得截图。FlashPick 这种“One-Tap / Double-Tap”的极简交互，完美契合了当下“懒人经济”与“碎片化阅读”的大趋势。

## 6. 其他

### 🔹 如果你还有其他的创新想法，也请一并告诉我们。这也许会成为我们录取的重要原因
**全局屏幕高亮 (Universal Screen Highlighter)**
基于我之前 **PaperMind** 的开发经验——利用 LLM 实时针对用户标记部分解析论文逻辑，模拟用户意图，生成结构化笔记——我产生了一个更激进的想法：**打破 App 之间的围墙，让用户像在 PDF 上做高亮一样，在任意 App（如微信公众号、知乎、Twitter）中随意涂画高亮。**
FlashPick 目前是通过“双击回溯”来捕捉即时灵感，而“全局高亮”将赋予用户更极致的主动权——**完全由用户手动定义什么是“焦点”**。FlashPick 可以利用无障碍服务 (Accessibility) 或视觉识别，将这些“跨应用的高亮”持久化保存，并建立索引。这意味着，**整个手机操作系统将变成一本可被用户随意标注的“大书”**，这将彻底重构移动端的信息消费方式。

### 🔹 请推荐过去一年读的最有意思的一本书或者一篇论文
**《以交易为生》 (Trading for a Living)**
虽是金融类书籍，但它对我创业影响极深。书中最核心的观点是**“3M 法则” (Mind, Method, Money)**，尤其是 Mind（心理）：交易员必须克服贪婪与恐惧，**在任何时候保持冷静，有计划地行事，决不能让情绪影响操作**。
这种**“反人性”的纪律性**，与创业极其相似——在市场波动（User Feedback）面前，创始人不能被情绪左右，而应像冷酷的交易员一样，基于数据（Data-Driven）坚定执行策略。

### 🔹 请告诉我们一件你们发现的奇妙或有趣的事情
**人类与 LLM 的“智力倒挂”现象。**
现在的强 LLM（如 GPT-4）本质上是一个超级强大的 Knowledge Base，在知识广度上甚至已经超越人类。但奇怪的是，人类依然是那个能通过交流、调研、思考来产生富有 Insight 的 **Non-trivial Idea** 的主体，而 LLM 目前很难做到这一点。
我经常思考这背后的原因：也许是因为 LLM 缺失了 **Long Horizon** 的规划能力，以及 **Online Lifelong Learning**（终身在线学习）与外界真实环境持续交互、反馈、修正的过程。这正是我在 FlashPick 中试图探索的方向——让 AI 不仅仅是“读过书”的学霸，更是能通过长期陪伴用户、感知环境变迁而产生“智慧”的 Agent。
